Тестирование профилей безопасности.

Осуществляется только на эталонной системе, с учетом всех функциональных ограничений программного обеспечения.

Важно! Если в процессе проверки поведение программы не соответствует ожидаемому, загрузить профиль в режиме complain, после чего повторить проверку предварительно удалив все временные файлы и файлы настроек, созданные в процессе предыдущего запуска. Для подтверждения дефекта программы, проверку следует проводить с полностью выключенной подсистемой AppArmor (параметр ядра во время загрузки установить в 0.



Общие методы проверок.

Проверка динамических библиотек:
1) Проверяем все зависимые программы
# equery d <имя пакета>


Проверка приложений с CLI:
1) Вывод сообщения в консоль с отображение помощи (подсказки).


Проверка приложений с GUI:
1) Предварительно удалить (или временно переименовать) ранее созданные настройки прогарммы. Проверить работу прогарммы с созданием настроек и их сохранением, а так же работу с настройками по умолчания.
2) Открытие/сохранение файлов.
3) Открытие/отображение помощи.
4) Открытие внешних программ (переход на страницу проекта в интернете, отсылка email авторам).
5) Если предусмотрена работа через mime, проверить запуск программы через двойной клик на файле привязанного к программе типа.
6) Проверить запуск второй копии программы (учесть механизм отслеживания запуска только одной копии программы, если он есть).
